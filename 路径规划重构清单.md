# 路径规划重构清单

## 当前最严重的问题

1. 现在很多景区/校园的路网数据仍然来自导入脚本生成的网格假路网，和真实底图不是同一套数据源。
2. 这会导致“地图底图是真实 OSM，但导航点和道路是虚构的”，所以路线、起终点、设施位置会出现错位。
3. 前端已经能画出路线，但如果底层路网不真实，前端再优化也只能缓解显示问题，不能根治“地图对不上”。

## 本次重构目标

1. 路线必须尽量基于真实地图数据，而不是继续依赖规则网格。
2. 支持从当前位置出发，到单个目标点的最优路径。
3. 支持输入多个目标点，生成途经多点的最优路径。
4. 支持参观后返回起点的闭环路线。
5. 地图界面必须清楚展示起点、终点、途经点、回程段和行进方向。
6. 当真实设施位置与可规划路网节点不重合时，地图要同时展示两者，并给出映射引导。

## 已完成

### 前端交互层

1. 已支持单终点导航与多目标点游览两种模式切换。
2. 已支持多目标点最优路径。
3. 已支持返回起点的闭环路线。
4. 已支持使用当前位置作为起点。
5. 已支持从查询页一键把设施带入路径规划页作为终点。
6. 已支持多段路线总览、分段着色和回程段样式。

### 地图表达层

1. 已支持起点、终点、途经点、返回起点的不同标记。
2. 已支持路线方向箭头。
3. 已支持真实设施位置与导航节点之间的虚线映射。
4. 已支持起终点重叠时的偏移显示，减少标记遮挡。

## 接下来必须做

### A. 真实地图数据重构

1. 停止把真实景区/校园继续导入成规则网格路网。
2. 为数据导入新增“真实地图模板”方案。
3. 第一阶段优先落地一个真实可用的共享地图模板。
4. 共享模板优先考虑北京邮电大学及其周边真实 OSM 路网。
5. 模板中需要包含：
   - 真实道路
   - 真实建筑
   - 真实服务设施
   - 足够的节点数与边数
6. 导入脚本改成：
   - 先读取真实 OSM 道路和建筑
   - 再映射成项目内的 `RoadGraphNode` / `RoadGraphEdge`
   - 再把景点、楼宇、设施挂接到真实路网节点附近
7. 第二阶段再逐步扩展为“每个景区/校园各自一套真实地图”。

### B. 路径规划准确性

1. 优先使用真实道路几何进行路线展示。
2. 起点和终点若不在路网上，要展示真实位置与映射节点。
3. 查询页、景区详情页、一日计划页进入路径规划时，都要把来源地点自动带入。
4. 多目标点模式下，需要把每一段的起终点都清楚标出来。

### C. 界面表达

1. 继续优化地图图例，让“真实位置”和“导航节点”的区别一眼能看懂。
2. 继续优化方向箭头密度，避免路线太长时箭头太少。
3. 给结果面板增加“真实地点是否已映射到路网”的提示。

## 默认执行顺序

1. 先完成真实地图数据导入方案设计与脚本改造。
2. 再把共享真实地图模板接入当前 200 个景区/校园。
3. 然后继续扩展到“部分高频景区/校园使用各自真实地图”。
4. 最后再做地图细节与交互精修。
