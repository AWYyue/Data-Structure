# 榜单与封面优化清单

## 当前已确认的问题

1. 榜单排序字段与前端展示字段不一致  
   热度榜此前按 `popularity` 排序，但页面展示更醒目的是 `visitorCount`，会让用户感觉 `TOP 1` 不是最高。  
   评分榜此前按 `averageRating` 排序，但页面有时展示 `rating`，也会造成观感不一致。

2. 景区总览承载量不够  
   当前数据库实际已有 `200` 个景区/校园，但总览页此前只取了前 `24` 个，无法体现完整数据规模。

3. 部分封面质量不稳定  
   虽然已经接入 Wikimedia Commons 和主题图兜底，但仍有部分条目出现：
   - 图像构图不适合做横幅封面
   - 校门、主楼、远景优先级不够高
   - 个别景区仍需人工白名单修正

## 下一步按顺序执行

1. 统一后端榜单字段语义  
   目标：让 `popularity / averageRating / reviewCount` 成为唯一榜单主排序字段，并减少 `rating / visitorCount` 对榜单认知的干扰。

2. 给景区总览增加更多筛选和分页能力  
   目标：支持按城市、类别、景区/校园筛选，并支持跳页或更长距离浏览。

3. 扫描全部 `200` 个条目的封面质量  
   目标：生成“低质量封面待修复名单”，优先处理首页高频出现的前 `50` 个。

4. 建立校园封面优先规则  
   目标：优先选择校门、主楼、图书馆、校园远景，尽量避免标识墙、局部构件、宣传牌类图片。

5. 建立景区封面优先规则  
   目标：优先选择主景观、地标建筑、自然远景，尽量避免局部细节、游客随拍、说明牌类图片。

6. 补一批手工白名单  
   目标：对北京交通大学这类高频条目继续人工指定更好的 Commons 图片，逐步覆盖首页与榜单常驻条目。

7. 给封面增加质量标签与来源说明  
   目标：后续可以区分“真实景区图 / 主题兜底图 / 手工优选图”，方便演示和维护。

## 建议优先级

1. 先做首页和总览页高频封面前 `50` 名优化
2. 再做全量 `200` 条低质量封面扫描
3. 最后再补更细的筛选、分页和来源展示
