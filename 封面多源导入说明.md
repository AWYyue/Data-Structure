# 封面多源导入说明

当前项目的封面导入脚本已经支持多图片源顺序回填。

默认顺序：

`wikimedia,pexels,unsplash`

含义：

1. 先尝试 `Wikimedia Commons`，优先保留更贴近真实景区/校园的封面。
2. 如果没有合适结果，并且配置了 `Pexels API Key`，再尝试 `Pexels`。
3. 如果仍然没有合适结果，并且配置了 `Unsplash Access Key`，再尝试 `Unsplash`。

## 环境变量

可选环境变量：

1. `PEXELS_API_KEY`
2. `UNSPLASH_ACCESS_KEY`
3. `UNSPLASH_APP_NAME`
4. `COVER_PROVIDER_ORDER`
5. `WIKIMEDIA_TARGET_NAMES`
6. `WIKIMEDIA_FORCE_UPDATE`

说明：

1. 没有配置 `PEXELS_API_KEY` 时，会自动跳过 `Pexels`。
2. 没有配置 `UNSPLASH_ACCESS_KEY` 时，会自动跳过 `Unsplash`。
3. `COVER_PROVIDER_ORDER` 可以自定义顺序，例如：
   `pexels,unsplash,wikimedia`
4. `WIKIMEDIA_TARGET_NAMES` 可用于只更新指定景区/校园，多个名称用英文逗号分隔。
5. `WIKIMEDIA_FORCE_UPDATE=true` 可强制覆盖已有封面。

## 常用命令

在 `backend` 目录执行：

```powershell
npm run import-covers:v2
```

只更新指定条目：

```powershell
$env:WIKIMEDIA_TARGET_NAMES='北京邮电大学,湖北省博物馆'
npm run import-covers:v2
```

优先使用更高质感图片源：

```powershell
$env:COVER_PROVIDER_ORDER='pexels,unsplash,wikimedia'
$env:PEXELS_API_KEY='你的 Pexels Key'
$env:UNSPLASH_ACCESS_KEY='你的 Unsplash Key'
npm run import-covers:v2
```

## 当前建议

建议默认继续使用：

`wikimedia,pexels,unsplash`

原因：

1. `Wikimedia Commons` 更容易命中真实景区/校园本体。
2. `Pexels / Unsplash` 更适合给缺图或差图条目补高质量封面。
3. 这样可以兼顾“真实性”和“视觉质感”。
